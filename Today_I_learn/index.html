Best Practices

So you're trying to be a full-stack web developer?

Here are some tips I gathered from the web, that might be helpful as a beginner!

In full stack web dev:

  • Master one or two of the core technologies!
    • With the amount of technologies available, trying to master all of them would be almost impossible to do.
  • Chat with experts!
    • It's always good to have an expert to consult with, after all experiences are better than theories.

Those two would definitely help you to not be overwhelmed by the amount of technologies there are for full-stack web developers. These next two tips are for your journey as a software engineer, should you want to transition to other software engineering branches.

In coding in general:

  • Comment and document! But avoid obvious comments!
    • Leaving comments to your code is always a good idea, specially when working with a team. Even if you're working alone, it is still a good idea to have comments so you will always know what a code block does without reading the whole code.
  • Consistent indentations!
    • Who doesn't want a pretty looking code block, right?

There are absolutely more things to consider and you'll learn them as you go along with your journey, but always remember these 4 tips and you'll soon become one of the best full-stack web developer out there.
